# Домашняя работа 4 — map, методы, структуры и память

Домашняя работа состоит из трёх крупных тематических разделов по 15 задач и трёх объединяющих задач. Условия в этом файле и TODO-комментарии описывают требуемое поведение полностью. Тесты проверяют условия, но не являются заменой описанию.

## Общие правила

- Не изменяй сигнатуры функций, названия типов и sentinel-ошибки.
- Не изменяй тесты, Makefile и workflow CI.
- Порядок элементов map не гарантирован. Там, где функция возвращает слайс ключей, требуемый порядок указан явно.
- Если условие требует сохранить данные без изменений, это относится ко всем полям объекта и вложенным коллекциям.
- Если функция возвращает map, для пустого результата она должна вернуть инициализированную пустую map, а не `nil`, если в строке задачи не указано обратное.
- Строки сравниваются точно: регистр, пробелы, пустые строки и Unicode-символы имеют значение.

# 01_maps — map и типовые backend-задачи

## Типы раздела

```go
type User struct {
    ID     int
    Name   string
    City   string
    Active bool
    Tags   []string
}

type Product struct {
    SKU      string
    Quantity int
    Price    int
}
```

| № | Функция | Вход | Выход и правила |
|---:|---|---|---|
| 1 | `CountWords(words []string) map[string]int` | Слайс слов. Может быть `nil`, пустым, содержать пустые строки, повторения и слова в разном регистре. | Вернуть новый счётчик `слово -> количество появлений`. Пустой вход даёт готовую к дальнейшей записи пустую map. `Go`, `go` и `GO` — разные ключи. |
| 2 | `GetOrDefault(values map[string]int, key string, fallback int) int` | Map чисел, ключ и запасное значение. Map может быть `nil`. | Вернуть значение ключа, если ключ присутствует. Если ключа нет — вернуть `fallback`. Существующий ключ со значением `0` не считается отсутствующим. |
| 3 | `DeleteAndReport(values map[string]int, key string) bool` | Изменяемая map и ключ. | Удалить ключ и вернуть `true`, если он существовал до вызова. Для отсутствующего ключа, пустой или `nil` map вернуть `false`. Остальные пары не менять. |
| 4 | `MergeCounters(left, right map[string]int) map[string]int` | Два счётчика. Любой из них может быть `nil`. | Вернуть новую map со всеми ключами. Для общих ключей сложить значения. Исходные map и результат не должны зависеть друг от друга после вызова. |
| 5 | `KeysByValueAtLeast(values map[string]int, min int) []string` | Map и минимальное допустимое значение. | Вернуть ключи, у которых значение `>= min`, в алфавитном порядке. Если подходящих ключей нет — вернуть пустой слайс. |
| 6 | `MaxKeyByValue(values map[string]int) (string, bool)` | Map чисел, включая отрицательные значения. | Вернуть ключ максимального значения и `true`. Для пустой map вернуть `"", false`. При одинаковом максимуме выбрать лексикографически меньший ключ. |
| 7 | `BuildUserIndex(users []User) map[int]User` | Слайс пользователей, ID могут повторяться. | Построить map `ID -> User`. При повторном ID оставить последнего пользователя из входного слайса. Пустой вход даёт пустую инициализированную map. |
| 8 | `GroupActiveUsersByCity(users []User) map[string][]User` | Слайс активных и неактивных пользователей. | Сгруппировать только активных пользователей по `City`. Порядок пользователей внутри города сохранить таким же, как во входном слайсе. Города без активных пользователей не добавлять. |
| 9 | `CountTags(users []User) map[string]int` | Пользователи со слайсами тегов. | Посчитать все появления каждого тега. Повтор одного тега у одного пользователя учитывается несколько раз. Пустой тег является обычным ключом. |
| 10 | `BuildInventory(products []Product) map[string]Product` | Слайс товаров, SKU могут повторяться. | Построить map `SKU -> Product`. При повторном SKU оставить последнюю запись целиком. Пустой вход даёт пустую инициализированную map. |
| 11 | `ReserveStock(inventory map[string]Product, sku string, count int) bool` | Склад значений, SKU и количество для резервирования. | Если товар существует, `count > 0` и остатка хватает, уменьшить `Quantity` и вернуть `true`. Иначе вернуть `false`, не меняя ни одно поле товара и остальные записи. |
| 12 | `Restock(inventory map[string]*Product, sku string, count int) bool` | Склад указателей, SKU и количество пополнения. | Если ключ существует, указатель не `nil` и `count > 0`, увеличить `Quantity` исходного объекта и вернуть `true`. В остальных случаях вернуть `false` без изменений. |
| 13 | `LowStockSKUs(inventory map[string]Product, limit int) []string` | Склад и порог остатка. | Вернуть SKU товаров с `Quantity <= limit` в алфавитном порядке. Отрицательные остатки и отрицательный limit обрабатываются обычным сравнением. |
| 14 | `InventoryValue(inventory map[string]Product) int` | Склад товаров. | Вернуть сумму `Quantity * Price` по всем записям. Для пустого и `nil` склада вернуть `0`. Значения не нормализуются: отрицательные числа участвуют в расчёте. |
| 15 | `ApplyPriceUpdates(inventory map[string]Product, updates map[string]int) int` | Склад и map `SKU -> новая цена`. | Обновить только существующие товары и только ценами `> 0`. Неизвестные SKU и неположительные цены пропустить. Вернуть количество реально обновлённых товаров. Остальные поля Product сохранить. |

# 02_methods — value receiver, pointer receiver и инварианты

## Типы и ошибки раздела

```go
type Account struct {
    ID       int
    Owner    string
    Balance  int
    Active   bool
    Currency string
}

type Cart struct {
    Items    map[string]int
    MaxItems int
}
```

В файле уже объявлены sentinel-ошибки. Возвращай именно их, чтобы вызывающий код мог проверять ошибку через `errors.Is`.

| № | Метод | Вход | Выход и правила |
|---:|---|---|---|
| 1 | `Account.Label() string` | Значение счёта. | Вернуть строку строго в формате `#ID Owner: Balance Currency`. Пример: `#1 Maria: 100 RUB`. Пустые поля не заменять текстом-заглушкой. |
| 2 | `Account.CanWithdraw(amount int) bool` | Счёт и сумма. | Вернуть `true`, только если счёт активен, `amount > 0` и `Balance >= amount`. Метод ничего не меняет. |
| 3 | `Account.Renamed(owner string) Account` | Счёт и новое имя владельца. | Вернуть копию со значением `Owner = owner`. Исходный счёт и остальные поля результата должны остаться прежними. |
| 4 | `(*Account).Rename(owner string) error` | Указатель на счёт и новое имя. | Изменить `Owner` исходного счёта. Для `nil` receiver вернуть `ErrNilAccount`. Пустое имя допустимо. Остальные поля не менять. |
| 5 | `(*Account).Deposit(amount int) error` | Указатель на счёт и сумма пополнения. | Для `nil` вернуть `ErrNilAccount`; для неактивного счёта — `ErrInactive`; для `amount <= 0` — `ErrInvalidAmount`. При успехе увеличить баланс и вернуть `nil`. При ошибке баланс не менять. |
| 6 | `(*Account).Withdraw(amount int) error` | Указатель на счёт и сумма списания. | Проверить `nil`, активность, положительную сумму и достаточный баланс. Использовать соответствующие sentinel-ошибки. При успехе уменьшить баланс, при ошибке оставить его прежним. |
| 7 | `(*Account).TransferTo(destination *Account, amount int) error` | Счёт-источник, счёт-получатель и сумма. | Ошибки: `ErrNilAccount`, `ErrNilDestination`, `ErrSameAccount`, `ErrInactive`, `ErrInvalidAmount`, `ErrInsufficientFunds`. При успехе уменьшить баланс источника и увеличить баланс получателя. При любой ошибке оба объекта должны остаться неизменными. |
| 8 | `(*Account).Activate() error` | Указатель на счёт. | Установить `Active = true`. Для `nil` вернуть `ErrNilAccount`. Повторная активация допустима. |
| 9 | `(*Account).Deactivate() error` | Указатель на счёт. | Установить `Active = false`. Для `nil` вернуть `ErrNilAccount`. Повторная деактивация допустима. |
| 10 | `Account.Snapshot() Account` | Значение счёта. | Вернуть независимую копию со всеми полями. Последующие изменения исходного значения не должны менять snapshot. |
| 11 | `(*Account).Reset()` | Указатель на счёт. | Сбросить существующий объект в zero value. Для `nil` receiver безопасно ничего не делать. |
| 12 | `(*Account).ApplyBonus(percent int) error` | Указатель на счёт и процент. | Для `nil` вернуть `ErrNilAccount`, для неактивного счёта — `ErrInactive`, для отрицательного процента — `ErrInvalidAmount`. Увеличить баланс на `Balance * percent / 100`; дробная часть отбрасывается. При ошибке баланс не менять. |
| 13 | `Account.SameOwner(other Account) bool` | Два счёта. | Вернуть результат точного сравнения `Owner`. Регистр и пробелы имеют значение. Остальные поля не учитывать. |
| 14 | `(*Cart).Add(sku string, count int) error` | Корзина, SKU и количество. | Для `nil` вернуть `ErrNilCart`; для пустого SKU — `ErrInvalidSKU`; для `count <= 0` — `ErrInvalidAmount`. После успешного добавления сумма всех количеств не должна превышать `MaxItems`, иначе вернуть `ErrCartLimit` без изменений. Map `Items` изначально может быть `nil`. |
| 15 | `Cart.TotalItems() int` | Корзина. | Вернуть сумму всех значений `Items`. Пустая или `nil` map даёт `0`. Значения суммируются как есть. |

# 03_structs — структуры в map, embedding и память

## Типы раздела

```go
type User struct {
    ID         int
    Name       string
    Active     bool
    LoginCount int
    Tags       []string
}

type Admin struct {
    User
    Permissions []string
}
```

В разделе также объявлены `Account`, `PremiumAccount`, `BadLayout` и `GoodLayout`.

| № | Функция или метод | Вход | Выход и правила |
|---:|---|---|---|
| 1 | `UpdateUserName(users map[int]User, id int, name string) bool` | Map пользователей, ID и имя. | Изменить `Name` только существующего пользователя и вернуть `true`. Если ключа нет — вернуть `false` и не создавать запись. Все остальные поля сохранить. |
| 2 | `ActivateUser(users map[int]User, id int) bool` | Map пользователей и ID. | Установить `Active = true` существующему пользователю. Если ключа нет — вернуть `false`. Остальные поля сохранить. |
| 3 | `IncrementLoginCount(users map[int]User, id int) bool` | Map пользователей и ID. | Увеличить `LoginCount` на единицу. Для отсутствующего пользователя вернуть `false`. Отрицательное начальное значение также увеличивается на один. |
| 4 | `CopyUsers(users map[int]User) map[int]User` | Map пользователей, у каждого может быть слайс `Tags`. | Вернуть глубокую независимую копию: изменение map, полей User или элементов `Tags` в результате не должно менять источник. Для пустого входа вернуть инициализированную пустую map. |
| 5 | `BuildPointerIndex(users []User) map[int]*User` | Слайс пользователей, ID могут повторяться. | Построить map указателей на независимые копии пользователей. При повторном ID оставить последнюю копию. Разные ID не должны указывать на один объект; изменения через индекс не должны менять входной слайс. |
| 6 | `RenameThroughPointer(users map[int]*User, id int, name string) bool` | Map указателей, ID и имя. | Если ключ существует и указатель не `nil`, изменить `Name` объекта и вернуть `true`. Иначе вернуть `false`. Пустое имя допустимо. |
| 7 | `User.DisplayName() string` | Пользователь. | Вернуть строку `ID:Name`, например `7:Maria`. |
| 8 | `Admin.DisplayName() string` | Администратор со встроенным User. | Вернуть строку `admin:ID:Name`. Метод должен иметь собственное поведение, отличное от `User.DisplayName`. |
| 9 | `Admin.HasPermission(permission string) bool` | Администратор и искомое право. | Вернуть `true`, если точная строка присутствует в `Permissions`. Пустая строка может быть обычным значением. |
| 10 | `RenameEmbeddedUser(admin *Admin, name string)` | Указатель на администратора и имя. | Изменить поле `Name` встроенного User. Для `nil` безопасно ничего не делать. ID, права и остальные поля сохранить. |
| 11 | `PremiumAccount.Total() int` | PremiumAccount со встроенным Account и Bonus. | Вернуть сумму `Balance + Bonus`, включая отрицательные значения. |
| 12 | `PremiumLabel(p PremiumAccount) string` | PremiumAccount. | Вернуть строку `ID:Balance total=Total`, например `7:100 total=130`. |
| 13 | `UserValueSize(user User) uintptr` | Любое значение User. | Вернуть фактический размер значения структуры в байтах. Содержимое строк и слайса не меняет размер самого заголовка структуры. |
| 14 | `UserPointerSize(user *User) uintptr` | Указатель на User, включая `nil`. | Вернуть размер переменной-указателя в байтах. Значение по адресу и `nil` не должны влиять на результат. |
| 15 | `LayoutSizes() (bad, good, saved uintptr)` | Аргументов нет. | Вернуть размеры `BadLayout`, `GoodLayout` и разницу `bad - good`. Для текущих типов `BadLayout` должен занимать больше памяти из-за расположения полей и padding. |

Для самостоятельной проверки расположения полей после выполнения задач можно запустить анализатор, указанный в `README.md`. Он не заменяет понимание причин padding.

# 04_common — объединяющие задачи

| № | Функция | Вход | Выход и правила |
|---:|---|---|---|
| 1 | `ApplyOperations(accounts map[int]*methods.Account, operations []Operation) LedgerResult` | Map счетов и последовательность операций `deposit`/`withdraw`. | Обрабатывать операции по порядку и не останавливаться после ошибки. Отсутствующий или `nil` счёт даёт `account ID: not found`; неизвестный `Kind` — `account ID: unknown operation "KIND"`; ошибки методов оборачиваются префиксом `account ID: `. В `Balances` вернуть итоговые балансы всех ненулевых счетов, в `Errors` — строки в порядке возникновения. Обе коллекции результата должны быть инициализированы. |
| 2 | `BuildAccountIndex(accounts []methods.Account) map[int]*methods.Account` | Слайс счетов, ID могут повторяться. | Построить map указателей на независимые копии. При повторном ID оставить последний счёт. Разные ключи не должны указывать на один объект, изменения индекса не должны менять входной слайс. |
| 3 | `ApplyUserEvents(users map[int]string, active map[int]bool, events []Event) (map[int]string, map[int]bool)` | Две map состояния и события. | Непустое `Name` обновляет имя, `Active != nil` обновляет статус. Пустые поля события не стирают прежние данные. События применяются по порядку, новые ID разрешены. Если входная map не `nil`, изменить и вернуть тот же объект; если `nil` — создать новую map. |
